Reinforcement Learning for Humanoid Control

Introduction to Reinforcement Learning in Robotics​

Reinforcement Learning (RL) has emerged as a powerful paradigm for learning complex control policies in robotics, particularly for humanoid robots that require sophisticated locomotion and manipulation skills. Unlike traditional control methods that rely on hand-designed controllers, RL enables robots to learn optimal behaviors through interaction with their environment, making it ideal for tasks that are difficult to engineer explicitly.

The RL Framework for Humanoid Control​

In the RL framework, a humanoid robot (the agent) interacts with its environment by taking actions and receiving rewards based on its performance. The goal is to learn a policy that maximizes cumulative rewards over time:

  • State (s): Robot's configuration, joint positions, velocities, sensor readings, and environmental information
  • Action (a): Joint torques, positions, or velocities to control the robot
  • Reward (r): Feedback signal that guides learning (e.g., forward velocity, balance maintenance, task completion)
  • Policy (π): Mapping from states to actions that the robot learns to optimize

RL Algorithms for Humanoid Control​

Proximal Policy Optimization (PPO)​

PPO is one of the most successful on-policy RL algorithms for humanoid control, known for its stability and sample efficiency:

import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np

class PPOActorCritic(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super(PPOActorCritic, self).__init__()

        # Actor network (policy)
        self.actor = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.Tanh(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.Tanh(),
            nn.Linear(hidden_dim, action_dim),
        )

        # Critic network (value function)
        self.critic = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.Tanh(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.Tanh(),
            nn.Linear(hidden_dim, 1),
        )

    def forward(self, state):
        action_mean = self.actor(state)
        value = self.critic(state)
        return action_mean, value

class PPOAgent:
    def __init__(self, state_dim, action_dim, lr=3e-4, clip_epsilon=0.2):
        self.actor_critic = PPOActorCritic(state_dim, action_dim)
        self.optimizer = optim.Adam(self.actor_critic.parameters(), lr=lr)
        self.clip_epsilon = clip_epsilon

    def select_action(self, state, deterministic=False):
        state_tensor = torch.FloatTensor(state).unsqueeze(0)
        action_mean, value = self.actor_critic(state_tensor)

        if deterministic:
            return action_mean.detach().numpy()[0], value.detach().numpy()[0]

        # Add Gaussian noise for exploration
        action_std = torch.ones_like(action_mean) * 0.1
        action = action_mean + torch.randn_like(action_mean) * action_std
        return action.detach().numpy()[0], value.detach().numpy()[0]

    def update(self, states, actions, old_log_probs, returns, advantages):
        states = torch.FloatTensor(states)
        actions = torch.FloatTensor(actions)
        old_log_probs = torch.FloatTensor(old_log_probs)
        returns = torch.FloatTensor(returns).unsqueeze(1)
        advantages = torch.FloatTensor(advantages)

        # Current policy
        new_action_means, new_values = self.actor_critic(states)
        new_log_probs = self.gaussian_log_prob(new_action_means, actions)

        # Policy ratio
        ratio = torch.exp(new_log_probs - old_log_probs)

        # PPO objective
        surr1 = ratio * advantages
        surr2 = torch.clamp(ratio, 1 - self.clip_epsilon, 1 + self.clip_epsilon) * advantages
        actor_loss = -torch.min(surr1, surr2).mean()

        # Value loss
        value_loss = nn.MSELoss()(new_values, returns)

        # Total loss
        total_loss = actor_loss + 0.5 * value_loss

        self.optimizer.zero_grad()
        total_loss.backward()
        self.optimizer.step()

    def gaussian_log_prob(self, means, samples):
        # Calculate log probability of Gaussian distribution
        log_probs = -0.5 * ((samples - means) ** 2).sum(dim=-1)
        return log_probs

Soft Actor-Critic (SAC)​

SAC is an off-policy algorithm that maximizes both reward and entropy, leading to more robust policies:

import torch
import torch.nn as nn
import torch.optim as optim
import numpy as np
import random
from collections import deque

class SACActor(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super(SACActor, self).__init__()

        self.net = nn.Sequential(
            nn.Linear(state_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
        )

        self.mean_layer = nn.Linear(hidden_dim, action_dim)
        self.log_std_layer = nn.Linear(hidden_dim, action_dim)

    def forward(self, state):
        x = self.net(state)
        mean = self.mean_layer(x)
        log_std = self.log_std_layer(x)
        log_std = torch.clamp(log_std, -20, 2)
        return mean, log_std

class SACCritic(nn.Module):
    def __init__(self, state_dim, action_dim, hidden_dim=256):
        super(SACCritic, self).__init__()

        self.q1 = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )

        self.q2 = nn.Sequential(
            nn.Linear(state_dim + action_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, 1)
        )

    def forward(self, state, action):
        sa = torch.cat([state, action], dim=1)
        q1 = self.q1(sa)
        q2 = self.q2(sa)
        return q1, q2

class SACAgent:
    def __init__(self, state_dim, action_dim, lr=3e-4, gamma=0.99, tau=0.005):
        self.actor = SACActor(state_dim, action_dim)
        self.critic = SACCritic(state_dim, action_dim)
        self.target_critic = SACCritic(state_dim, action_dim)

        # Copy critic weights to target critic
        self.target_critic.load_state_dict(self.critic.state_dict())

        self.actor_optimizer = optim.Adam(self.actor.parameters(), lr=lr)
        self.critic_optimizer = optim.Adam(self.critic.parameters(), lr=lr)

        self.gamma = gamma
        self.tau = tau
        self.target_entropy = -action_dim
        self.log_alpha = torch.zeros(1, requires_grad=True)
        self.alpha_optimizer = optim.Adam([self.log_alpha], lr=lr)

    def select_action(self, state, evaluate=False):
        state = torch.FloatTensor(state).unsqueeze(0)
        mean, log_std = self.actor(state)
        std = log_std.exp()

        if evaluate:
            return mean.detach().numpy()[0]

        # Reparameterization trick
        normal = torch.distributions.Normal(mean, std)
        x_t = normal.rsample()
        action = torch.tanh(x_t)
        log_prob = normal.log_prob(x_t) - torch.log(1 - action.pow(2) + 1e-6)
        log_prob = log_prob.sum(dim=1, keepdim=True)

        return action.detach().numpy()[0]

    def update(self, replay_buffer, batch_size=256):
        state, action, reward, next_state, done = replay_buffer.sample(batch_size)

        state = torch.FloatTensor(state)
        action = torch.FloatTensor(action)
        reward = torch.FloatTensor(reward).unsqueeze(1)
        next_state = torch.FloatTensor(next_state)
        done = torch.FloatTensor(done).unsqueeze(1)

        # Update Q functions
        with torch.no_grad():
            next_action, next_log_prob = self.actor(next_state)
            next_q1, next_q2 = self.target_critic(next_state, next_action)
            next_q = torch.min(next_q1, next_q2) - self.log_alpha * next_log_prob
            target_q = reward + (1 - done) * self.gamma * next_q

        current_q1, current_q2 = self.critic(state, action)
        q_loss = nn.MSELoss()(current_q1, target_q) + nn.MSELoss()(current_q2, target_q)

        self.critic_optimizer.zero_grad()
        q_loss.backward()
        self.critic_optimizer.step()

        # Update policy
        new_actions, log_prob = self.actor(state)
        q1, q2 = self.critic(state, new_actions)
        q = torch.min(q1, q2)
        policy_loss = (self.log_alpha * log_prob - q).mean()

        self.actor_optimizer.zero_grad()
        policy_loss.backward()
        self.actor_optimizer.step()

        # Update temperature
        alpha_loss = -(self.log_alpha * (log_prob + self.target_entropy).detach()).mean()
        self.alpha_optimizer.zero_grad()
        alpha_loss.backward()
        self.alpha_optimizer.step()

        # Update target networks
        for target_param, param in zip(self.target_critic.parameters(), self.critic.parameters()):
            target_param.data.copy_(self.tau * param.data + (1 - self.tau) * target_param.data)

Creating RL Training Environments​

Isaac Sim provides comprehensive tools for creating RL training environments for humanoid robots:

# Isaac Sim RL environment for humanoid locomotion
import omni
from omni.isaac.core import World
from omni.isaac.core.utils.stage import add_reference_to_stage
from omni.isaac.core.utils.prims import get_prim_at_path
from omni.isaac.core.robots import Robot
from omni.isaac.core.tasks import BaseTask
from pxr import Gf
import numpy as np

class HumanoidLocomotionTask(BaseTask):
    def __init__(
        self,
        name,
        offset=None,
        target_velocity=1.0,
        max_episode_length=1000
    ):
        super().__init__(name=name, offset=offset)
        self._num_envs = 1
        self._target_velocity = target_velocity
        self._max_episode_length = max_episode_length
        self._current_step = 0

    def set_up_scene(self, scene) -> None:
        # Add ground plane
        super().set_up_scene(scene)
        scene.add_default_ground_plane()

        # Add humanoid robot
        self._robot = Robot(
            prim_path="/World/Robot",
            name="humanoid_robot",
            usd_path="/path/to/humanoid_model.usd",
            position=np.array([0.0, 0.0, 1.0])
        )
        scene.add(self._robot)

    def get_observations(self) -> dict:
        # Get robot state information
        positions = self._robot.get_world_poses(clone=True)[0]
        orientations = self._robot.get_world_poses(clone=True)[1]
        joint_positions = self._robot.get_joint_positions()
        joint_velocities = self._robot.get_joint_velocities()

        # Create observation vector
        observation = {
            "joint_positions": joint_positions,
            "joint_velocities": joint_velocities,
            "base_position": positions[0],
            "base_orientation": orientations[0],
            "target_velocity": self._target_velocity
        }

        return observation

    def pre_physics_step(self, actions) -> None:
        # Apply actions to robot
        if actions is not None:
            self._robot.set_joint_position_targets(actions)

    def get_rewards(self) -> dict:
        # Calculate reward based on forward velocity and stability
        current_velocity = self._calculate_forward_velocity()
        stability_penalty = self._calculate_stability_penalty()

        # Reward for forward motion
        velocity_reward = current_velocity * 10.0

        # Penalty for falling
        fall_penalty = self._check_fall() * -100.0

        # Penalty for instability
        stability_reward = -stability_penalty * 5.0

        total_reward = velocity_reward + stability_reward + fall_penalty
        return {"total_reward": total_reward}

    def is_done(self) -> dict:
        # Check if episode is done
        done = self._check_fall() or (self._current_step >= self._max_episode_length)
        return {"done": done}

    def _calculate_forward_velocity(self):
        # Calculate robot's forward velocity
        current_pos = self._robot.get_world_poses(clone=True)[0][0]
        if hasattr(self, '_prev_pos'):
            velocity = (current_pos[0] - self._prev_pos[0]) / self._world.get_physics_dt()
            self._prev_pos = current_pos
            return velocity
        else:
            self._prev_pos = current_pos
            return 0.0

    def _calculate_stability_penalty(self):
        # Calculate penalty for unstable posture
        orientation = self._robot.get_world_poses(clone=True)[1][0]
        # Check if robot is upright (simplified)
        up_vector = Gf.Quatf(orientation[6], orientation[3], orientation[4], orientation[5]).Rotate(Gf.Vec3f(0, 0, 1))
        stability = abs(up_vector[2] - 1.0)  # 0 when upright, higher when tilted
        return stability

    def _check_fall(self):
        # Check if robot has fallen
        base_pos = self._robot.get_world_poses(clone=True)[0][0]
        return base_pos[2] < 0.5  # Fallen if base is too low

Integration with RL Libraries​

Isaac Sim can be integrated with popular RL libraries like Stable-Baselines3:

from stable_baselines3 import PPO
from stable_baselines3.ppo import MlpPolicy
from stable_baselines3.common.env_util import make_vec_env
from stable_baselines3.common.callbacks import EvalCallback
import gym
from gym import spaces

class IsaacHumanoidEnv(gym.Env):
    def __init__(self):
        super(IsaacHumanoidEnv, self).__init__()

        # Define action and observation spaces
        self.action_space = spaces.Box(
            low=-1.0, high=1.0, shape=(19,), dtype=np.float32  # 19 joint torques
        )
        self.observation_space = spaces.Box(
            low=-np.inf, high=np.inf, shape=(60,), dtype=np.float32  # State vector
        )

        # Initialize Isaac Sim world
        self.world = World(stage_units_in_meters=1.0)
        self.task = HumanoidLocomotionTask(name="humanoid_task")
        self.world.add_task(self.task)
        self.world.reset()

        self.episode_step = 0
        self.max_episode_steps = 1000

    def reset(self):
        self.world.reset()
        self.episode_step = 0
        obs = self._get_observation()
        return obs

    def step(self, action):
        # Apply action to robot
        self.world.step(render=True)
        self.task.pre_physics_step(action)

        # Get observation and reward
        obs = self._get_observation()
        reward = self._get_reward()
        done = self._is_done()

        self.episode_step += 1
        info = {}

        return obs, reward, done, info

    def _get_observation(self):
        # Get current robot state as observation
        obs_dict = self.task.get_observations()
        # Flatten and normalize observation
        obs = np.concatenate([
            obs_dict["joint_positions"],
            obs_dict["joint_velocities"],
            obs_dict["base_position"],
            obs_dict["base_orientation"]
        ])
        return obs

    def _get_reward(self):
        # Get reward from task
        rewards = self.task.get_rewards()
        return rewards["total_reward"]

    def _is_done(self):
        # Check if episode is done
        done_dict = self.task.is_done()
        return done_dict["done"] or self.episode_step >= self.max_episode_steps

# Train the humanoid robot using Isaac Sim
def train_humanoid():
    # Create Isaac Sim environment
    env = IsaacHumanoidEnv()

    # Create PPO agent
    model = PPO(
        MlpPolicy,
        env,
        verbose=1,
        tensorboard_log="./logs/",
        learning_rate=3e-4,
        n_steps=2048,
        batch_size=64,
        n_epochs=10,
        gamma=0.99,
        gae_lambda=0.95,
        clip_range=0.2,
        ent_coef=0.01
    )

    # Train the model
    model.learn(total_timesteps=1000000)

    # Save the trained model
    model.save("humanoid_policy")

    # Test the trained policy
    obs = env.reset()
    for i in range(1000):
        action, _states = model.predict(obs)
        obs, rewards, dones, info = env.step(action)
        if dones:
            obs = env.reset()

Sim-to-Real Transfer for RL Policies​

Domain Randomization​

Domain randomization is crucial for successful Sim-to-Real transfer:

class DomainRandomization:
    def __init__(self, env):
        self.env = env
        self.randomization_params = {
            'mass_range': [0.8, 1.2],  # 80% to 120% of original mass
            'friction_range': [0.5, 1.5],  # Friction coefficient range
            'actuator_range': [0.9, 1.1],  # Actuator strength range
            'sensor_noise_range': [0.001, 0.01],  # Sensor noise range
        }

    def randomize_environment(self):
        """Randomize environment parameters for domain randomization"""
        # Randomize robot masses
        self.randomize_masses()

        # Randomize friction coefficients
        self.randomize_friction()

        # Randomize actuator properties
        self.randomize_actuators()

        # Randomize sensor noise
        self.randomize_sensors()

    def randomize_masses(self):
        """Randomize link masses"""
        for link in self.env.robot.links:
            original_mass = link.mass
            random_factor = np.random.uniform(
                self.randomization_params['mass_range'][0],
                self.randomization_params['mass_range'][1]
            )
            link.mass = original_mass * random_factor

    def randomize_friction(self):
        """Randomize friction coefficients"""
        for joint in self.env.robot.joints:
            random_factor = np.random.uniform(
                self.randomization_params['friction_range'][0],
                self.randomization_params['friction_range'][1]
            )
            joint.friction = joint.original_friction * random_factor

    def apply_randomization(self, step):
        """Apply randomization periodically during training"""
        if step % 1000 == 0:  # Randomize every 1000 steps
            self.randomize_environment()

System Identification and Model Adaptation​

class SystemIdentifier:
    def __init__(self, sim_env, real_robot):
        self.sim_env = sim_env
        self.real_robot = real_robot
        self.sim_params = {}
        self.real_params = {}
        self.adaptation_model = None

    def identify_system(self):
        """Identify system parameters from real robot data"""
        # Collect data from real robot
        real_data = self.collect_real_data()

        # Identify parameters
        self.real_params = self.estimate_parameters(real_data)

        # Update simulation to match real robot
        self.update_simulation_params()

        return self.real_params

    def collect_real_data(self):

        data = []

        for input_sequence in self.generate_excitation_signals():
            # Apply input and collect response
            response = self.real_robot.apply_input(input_sequence)

        return data

    def estimate_parameters(self, data):
        """Estimate system parameters from collected data"""
        # Use system identification techniques
        # (e.g., least squares, maximum likelihood, etc.)

        estimated_params = {}

        # Estimate mass properties
        estimated_params['mass'] = self.estimate_mass_properties(data)

Advanced RL Techniques for Humanoid Control​

Hierarchical RL​

Hierarchical RL can break down complex humanoid tasks into manageable sub-tasks:

class HierarchicalRL:
    def __init__(self, low_level_agent, high_level_agent):
        self.low_level_agent = low_level_agent  # Motor control
        self.high_level_agent = high_level_agent  # Task planning
        self.subtask_duration = 100  # Steps per subtask

    def get_action(self, state, step):
        # High-level agent selects subtask every N steps
        if step % self.subtask_duration == 0:
            self.current_subtask = self.high_level_agent.select_subtask(state)

        # Low-level agent executes subtask-specific policy
        action = self.low_level_agent.execute_subtask(
            state, self.current_subtask
        )

        return action

Curriculum Learning​

Gradually increasing task difficulty during training:

class CurriculumLearning:
    def __init__(self):
        self.current_level = 0
        self.difficulty_thresholds = [0.5, 0.7, 0.85, 1.0]  # Performance thresholds
        self.curriculum_tasks = [
            'stand_still',
            'walk_forward',
            'walk_turn',
            'navigate_obstacles',
            'complex_maneuvers'
        ]

    def update_curriculum(self, performance):
        """Update curriculum based on performance"""
        if (self.current_level < len(self.difficulty_thresholds) and
            performance >= self.difficulty_thresholds[self.current_level]):
            self.current_level += 1
            print(f"Advancing to curriculum level: {self.curriculum_tasks[self.current_level]}")

        return self.curriculum_tasks[self.current_level]

Isaac Sim RL Best Practices​

1. Reward Shaping​

Design reward functions that guide learning effectively:

def design_reward_function(robot_state, target_state, action):
    """Well-designed reward function for humanoid control"""
    reward = 0.0

    # Forward velocity reward
    forward_vel = calculate_forward_velocity(robot_state)
    reward += forward_vel * 10.0

    # Energy efficiency penalty
    energy_cost = np.sum(np.abs(action)) * 0.01
    reward -= energy_cost

    # Stability reward (keeping torso upright)
    torso_orientation = get_torso_orientation(robot_state)
    stability_bonus = max(0, torso_orientation[2]) * 5.0  # Z-axis up
    reward += stability_bonus

    # Smoothness penalty (action differences)
    if hasattr(self, 'prev_action'):
        action_smoothness = np.sum(np.abs(action - self.prev_action)) * 0.1
        reward -= action_smoothness
    self.prev_action = action

    # Survival bonus (staying upright)
    if not is_fallen(robot_state):
        reward += 1.0

    return reward

Challenges and Solutions in RL for Humanoid Control​

1. Sample Efficiency​

  • Use domain randomization to improve generalization
  • Implement curriculum learning for gradual skill acquisition
  • Leverage prior knowledge through imitation learning

2. Safety During Training​

  • Implement safety constraints in simulation
  • Use safe exploration techniques
  • Monitor and limit joint limits and forces

3. Sim-to-Real Transfer​

  • Carefully calibrate simulation parameters
  • Use system identification techniques
  • Validate policies in real-world conditions gradually

Reinforcement Learning provides a powerful approach to learning complex humanoid control policies, enabling robots to acquire sophisticated locomotion and manipulation skills through interaction with their environment. When combined with Isaac Sim's high-fidelity simulation capabilities, RL can significantly accelerate the development of capable humanoid robots.
