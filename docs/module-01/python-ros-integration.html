Writing ROS 2 Nodes in Python (rclpy)

Introduction to rclpy​

rclpy is the Python client library for ROS 2. It provides a Python API that allows you to create ROS 2 nodes, publish and subscribe to topics, provide and use services, and manage parameters. This library is the Python equivalent of rclcpp for C++.

Setting Up Your Python Environment​

Before creating ROS 2 nodes in Python, ensure your environment is properly configured:

# Source your ROS 2 installation
source /opt/ros/humble/setup.bash  # Replace 'humble' with your ROS 2 distribution

# Create a workspace (if you haven't already)
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws

# Source your workspace (if you have one)
source install/setup.bash

Basic Node Structure​

Every ROS 2 Python node follows the same basic structure:

import rclpy
from rclpy.node import Node

class MyNode(Node):
    def __init__(self):
        super().__init__('node_name')
        # Node initialization code here

def main(args=None):
    rclpy.init(args=args)
    node = MyNode()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()

if __name__ == '__main__':
    main()

Creating a Publisher Node​

A publisher node sends messages to a topic. Here's a complete example of a publisher node that publishes string messages:

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class MinimalPublisher(Node):
    def __init__(self):
        super().__init__('minimal_publisher')
        self.publisher_ = self.create_publisher(String, 'topic', 10)
        timer_period = 0.5  # seconds
        self.timer = self.create_timer(timer_period, self.timer_callback)
        self.i = 0

    def timer_callback(self):
        msg = String()
        msg.data = f'Hello World: {self.i}'
        self.publisher_.publish(msg)
        self.get_logger().info(f'Publishing: "{msg.data}"')
        self.i += 1


def main(args=None):
    rclpy.init(args=args)

    minimal_publisher = MinimalPublisher()

    rclpy.spin(minimal_publisher)

    # Destroy the node explicitly
    minimal_publisher.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()

Key Components of a Publisher Node:​

  1. Import Statements: Import rclpy, the Node class, and the message type (std_msgs.msg.String)
  2. Node Class: Inherit from Node and initialize with a unique node name
  3. Publisher Creation: Use create_publisher() to create a publisher for a specific topic and message type
  4. Timer: Use a timer to periodically publish messages
  5. Message Creation: Create and populate message objects with data
  6. Publish: Call publish() to send the message to the topic

Creating a Subscriber Node​

A subscriber node receives messages from a topic. Here's a complete example:

#!/usr/bin/env python3

import rclpy
from rclpy.node import Node
from std_msgs.msg import String


class MinimalSubscriber(Node):
    def __init__(self):
        super().__init__('minimal_subscriber')
        self.subscription = self.create_subscription(
            String,
            'topic',
            self.listener_callback,
            10)
        self.subscription  # prevent unused variable warning

    def listener_callback(self, msg):
        self.get_logger().info(f'I heard: "{msg.data}"')


def main(args=None):
    rclpy.init(args=args)

    minimal_subscriber = MinimalSubscriber()

    rclpy.spin(minimal_subscriber)

    # Destroy the node explicitly
    minimal_subscriber.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()

Key Components of a Subscriber Node:​

  1. Import Statements: Import rclpy, the Node class, and the message type
  2. Node Class: Inherit from Node and initialize with a unique node name
  3. Subscription Creation: Use create_subscription() to subscribe to a topic with a callback function
  4. Callback Function: Define a function that will be called when a message is received
  5. Message Processing: Process the received message in the callback

Running Publisher and Subscriber Nodes​

Method 1: Using ros2 run​

First, make sure your Python file is executable:

chmod +x my_package/my_publisher.py
chmod +x my_package/my_subscriber.py

Then run each node in separate terminals:

Terminal 1 (Publisher):

ros2 run my_package my_publisher

Terminal 2 (Subscriber):

ros2 run my_package my_subscriber

Method 2: Direct Python Execution​

python3 my_publisher.py
python3 my_subscriber.py

Advanced Publisher Features​

Publishing Different Message Types​

import rclpy
from rclpy.node import Node
from std_msgs.msg import Int32, Float32
from geometry_msgs.msg import Twist


class AdvancedPublisher(Node):
    def __init__(self):
        super().__init__('advanced_publisher')

        # Multiple publishers for different topics
        self.int_publisher = self.create_publisher(Int32, 'integer_topic', 10)
        self.float_publisher = self.create_publisher(Float32, 'float_topic', 10)
        self.cmd_vel_publisher = self.create_publisher(Twist, 'cmd_vel', 10)

        self.timer = self.create_timer(1.0, self.publish_messages)
        self.counter = 0

    def publish_messages(self):
        # Publish integer
        int_msg = Int32()
        int_msg.data = self.counter
        self.int_publisher.publish(int_msg)

        # Publish float
        float_msg = Float32()
        float_msg.data = float(self.counter) * 0.5
        self.float_publisher.publish(float_msg)

        # Publish Twist (velocity command)
        twist_msg = Twist()
        twist_msg.linear.x = 1.0  # Move forward
        twist_msg.angular.z = 0.5  # Rotate
        self.cmd_vel_publisher.publish(twist_msg)

        self.get_logger().info(f'Published: {self.counter}, {float(self.counter) * 0.5}')
        self.counter += 1


def main(args=None):
    rclpy.init(args=args)
    node = AdvancedPublisher()
    rclpy.spin(node)
    node.destroy_node()
    rclpy.shutdown()


if __name__ == '__main__':
    main()

Best Practices for Python ROS 2 Nodes​

1. Error Handling​

def main(args=None):
    try:
        rclpy.init(args=args)
        node = MyNode()
        rclpy.spin(node)
    except KeyboardInterrupt:
        pass
    finally:
        node.destroy_node()
        rclpy.shutdown()

2. Logging​

# Use the built-in logger
self.get_logger().info('Information message')
self.get_logger().warn('Warning message')
self.get_logger().error('Error message')
self.get_logger().debug('Debug message')

3. Proper Node Destruction​

def destroy_node(self):
    # Clean up resources before destroying the node
    # Cancel timers, destroy publishers/subscribers, etc.
    super().destroy_node()

Common Message Types​

Here are some commonly used message types in ROS 2:

  • std_msgs.msg.String - For string data
  • std_msgs.msg.Int32, std_msgs.msg.Float32 - For numeric data
  • sensor_msgs.msg.Image - For image data
  • geometry_msgs.msg.Twist - For velocity commands
  • geometry_msgs.msg.Pose - For position and orientation
  • nav_msgs.msg.Odometry - For odometry data

Troubleshooting Common Issues​

  1. Node not found: Ensure the package is built and sourced
  2. Import errors: Check that the message types are properly imported
  3. Topic not connecting: Verify topic names match between publisher and subscriber
  4. Permission errors: Make sure your Python files have execute permissions
