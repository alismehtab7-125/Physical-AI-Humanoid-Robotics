Introduction to ROS 2

What is ROS 2?​

ROS 2 (Robot Operating System 2) is not an operating system, despite its name. Rather, it's a flexible framework for writing robot software that provides services designed for a heterogeneous computer cluster. It includes hardware abstraction, device drivers, libraries, visualizers, message-passing, package management, and more.

ROS 2 is the next generation of the Robot Operating System, designed to address the limitations of ROS 1 and provide a more robust, secure, and scalable platform for robotics development.

ROS 1 vs ROS 2: Key Differences​

ROS 2 was developed to overcome several limitations of ROS 1:

Architecture​

  • ROS 1: Built on a centralized architecture using a single master node
  • ROS 2: Uses a distributed architecture based on DDS (Data Distribution Service)

Communication​

  • ROS 1: Uses TCPROS/UDPROS for communication
  • ROS 2: Uses DDS for communication, providing better reliability and performance

Security​

  • ROS 1: No built-in security features
  • ROS 2: Built-in security features including authentication, encryption, and access control

Real-time Support​

  • ROS 1: Limited real-time capabilities
  • ROS 2: Better real-time support with deterministic communication

Multi-robot Support​

  • ROS 1: Challenging to coordinate multiple robots
  • ROS 2: Designed for multi-robot systems with better coordination

Key Improvements in ROS 2​

DDS (Data Distribution Service)​

ROS 2 uses DDS as its communication middleware. DDS provides:

  • Reliable message delivery
  • Quality of Service (QoS) policies
  • Built-in discovery mechanisms
  • Language and platform independence
  • Real-time support

Quality of Service (QoS) Settings​

QoS policies allow you to configure how messages are delivered between nodes:

  • Reliability: Best effort or reliable delivery
  • Durability: Volatile or transient local data
  • History: Keep all messages or only the last few
  • Deadline: Maximum time between consecutive messages
  • Liveliness: How to detect if a publisher is alive

Security Features​

ROS 2 includes built-in security capabilities:

  • Authentication: Verify the identity of nodes
  • Encryption: Protect message contents
  • Access Control: Control which nodes can communicate
  • Logging: Track security-related events

The Robotic Nervous System Analogy​

Think of ROS 2 as the nervous system of a robot. Just as the nervous system allows different parts of the body to communicate and coordinate actions, ROS 2 enables different components of a robot to share information and work together.

Sensory Input​

Like sensory neurons in the nervous system, ROS 2 nodes can receive data from sensors such as cameras, LiDAR, IMUs, and other perception devices.

Information Processing​

Similar to how the brain processes sensory information, ROS 2 nodes can process, filter, and interpret data from various sources.

Motor Output​

Just as motor neurons control muscles, ROS 2 nodes can send commands to actuators, motors, and other control systems.

Coordination​

The nervous system coordinates complex behaviors; ROS 2 enables coordination between different robot subsystems like navigation, manipulation, and perception.

Getting Started with ROS 2​

ROS 2 provides the foundation for building complex robotic applications. Whether you're developing for autonomous vehicles, industrial robots, or research platforms, ROS 2 offers the tools and infrastructure needed to create sophisticated robotic systems.

The modular architecture of ROS 2 allows you to build upon existing packages and contribute your own, creating a rich ecosystem of reusable components that accelerate development and enable rapid prototyping.
