Launch Files and Parameter Management

Introduction to Launch Files​

Launch files in ROS 2 are used to start multiple nodes simultaneously with a single command. They provide a convenient way to configure and launch complex robotic systems with many interconnected nodes. Unlike ROS 1, ROS 2 uses Python-based launch files instead of XML, providing more flexibility and programmability.

Basic Launch File Structure​

A basic launch file follows this structure:

from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        # Node definitions go here
    ])

Creating a Simple Launch File​

Here's a basic launch file that starts both a publisher and subscriber node:

from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_package',
            executable='publisher_node',
            name='publisher_node',
            output='screen',
        ),
        Node(
            package='my_package',
            executable='subscriber_node',
            name='subscriber_node',
            output='screen',
        )
    ])

Node Parameters in Launch Files​

You can specify various parameters for each node in the launch file:

from launch import LaunchDescription
from launch_ros.actions import Node


def generate_launch_description():
    return LaunchDescription([
        Node(
            package='my_package',
            executable='talker',
            name='talker_node',
            parameters=[
                {'param_name': 'param_value'},
                {'frequency': 10.0},
            ],
            remappings=[
                ('original_topic', 'remapped_topic')
            ],
            output='screen',
        )
    ])

Advanced Launch File Features​

Using Launch Arguments​

Launch arguments allow you to pass parameters to your launch file at runtime:

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument
from launch.substitutions import LaunchConfiguration
from launch_ros.actions import Node


def generate_launch_description():
    # Declare launch arguments
    frequency_arg = DeclareLaunchArgument(
        'frequency',
        default_value='10.0',
        description='Publishing frequency'
    )

    # Get the value of the launch argument
    frequency = LaunchConfiguration('frequency')

    return LaunchDescription([
        frequency_arg,
        Node(
            package='my_package',
            executable='talker',
            name='talker_node',
            parameters=[
                {'frequency': frequency},
            ],
            output='screen',
        )
    ])

To run with custom arguments:

ros2 launch my_package my_launch_file.py frequency:=5.0

Conditional Launch​

You can conditionally launch nodes based on arguments:

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument, IfCondition
from launch.substitutions import LaunchConfiguration
from launch_ros.actions import Node


def generate_launch_description():
    # Declare launch arguments
    use_sim_time_arg = DeclareLaunchArgument(
        'use_sim_time',
        default_value='false',
        description='Use simulation time'
    )

    use_sim_time = LaunchConfiguration('use_sim_time')

    return LaunchDescription([
        use_sim_time_arg,
        Node(
            package='my_package',
            executable='talker',
            name='talker_node',
            parameters=[
                {'use_sim_time': use_sim_time},
            ],
            condition=IfCondition(use_sim_time),  # Only run if use_sim_time is true
            output='screen',
        )
    ])

Parameter Management with YAML Files​

Creating Parameter Files​

Parameter files in ROS 2 use YAML format and allow you to configure node parameters externally:

# my_params.yaml
talker_node:
  ros__parameters:
    frequency: 10.0
    message: "Hello from YAML"
    debug_mode: true
    thresholds:
      min_value: 0.5
      max_value: 10.0

Using Parameter Files in Launch Files​

from launch import LaunchDescription
from launch_ros.actions import Node
from ament_index_python.packages import get_package_share_directory
import os


def generate_launch_description():
    # Get the path to the parameter file
    params_file = os.path.join(
        get_package_share_directory('my_package'),
        'config',
        'my_params.yaml'
    )

    return LaunchDescription([
        Node(
            package='my_package',
            executable='talker',
            name='talker_node',
            parameters=[params_file],
            output='screen',
        )
    ])

Multiple Parameter Files​

You can use multiple parameter files:

def generate_launch_description():
    # Get paths to parameter files
    talker_params = os.path.join(
        get_package_share_directory('my_package'),
        'config',
        'talker_params.yaml'
    )

    listener_params = os.path.join(
        get_package_share_directory('my_package'),
        'config',
        'listener_params.yaml'
    )

    return LaunchDescription([
        Node(
            package='my_package',
            executable='talker',
            name='talker_node',
            parameters=[talker_params],
            output='screen',
        ),
        Node(
            package='my_package',
            executable='listener',
            name='listener_node',
            parameters=[listener_params],
            output='screen',
        )
    ])

Complex Launch File Example​

Here's a more complex example that demonstrates various launch file features:

from launch import LaunchDescription
from launch.actions import DeclareLaunchArgument, TimerAction
from launch.substitutions import LaunchConfiguration
from launch_ros.actions import Node
from ament_index_python.packages import get_package_share_directory
import os


def generate_launch_description():
    # Declare launch arguments
    use_sim_time_arg = DeclareLaunchArgument(
        'use_sim_time',
        default_value='false',
        description='Use simulation time'
    )

    robot_name_arg = DeclareLaunchArgument(
        'robot_name',
        default_value='robot1',
        description='Name of the robot'
    )

    # Get launch configurations
    use_sim_time = LaunchConfiguration('use_sim_time')
    robot_name = LaunchConfiguration('robot_name')

    # Get parameter file path
    params_file = os.path.join(
        get_package_share_directory('my_package'),
        'config',
        'robot_params.yaml'
    )

    # Define nodes
    talker_node = Node(
        package='my_package',
        executable='talker',
        name=[robot_name, '_talker'],
        parameters=[
            params_file,
            {'use_sim_time': use_sim_time}
        ],
        output='screen',
    )

    listener_node = Node(
        package='my_package',
        executable='listener',
        name=[robot_name, '_listener'],
        parameters=[
            params_file,
            {'use_sim_time': use_sim_time}
        ],
        output='screen',
    )

    # Timer to start the listener after 5 seconds
    delayed_listener = TimerAction(
        period=5.0,
        actions=[listener_node]
    )

    return LaunchDescription([
        use_sim_time_arg,
        robot_name_arg,
        talker_node,
        delayed_listener,
    ])

Running Launch Files​

Basic Launch Command​

ros2 launch my_package my_launch_file.py

With Arguments​

ros2 launch my_package my_launch_file.py use_sim_time:=true robot_name:=turtlebot

List Available Launch Files​

ros2 launch -s

Parameter Management Best Practices​

1. Organize Parameters by Node​

Structure your YAML files to group parameters by node:

# config/navigation_params.yaml
amcl_node:
  ros__parameters:
    use_sim_time: false
    alpha1: 0.2
    alpha2: 0.2
    alpha3: 0.2

bt_navigator_rclcpp_node:
  ros__parameters:
    use_sim_time: false
    global_frame: "map"
    robot_base_frame: "base_link"

2. Use Descriptive Parameter Names​

# Good
ros__parameters:
  sensor_frequency: 30.0
  max_velocity: 1.0
  safety_distance: 0.5

# Avoid
ros__parameters:
  freq: 30.0
  max_vel: 1.0
  dist: 0.5

3. Default Values and Documentation​

Document your parameters and provide sensible defaults:

# config/sensor_params.yaml
lidar_node:
  ros__parameters:
    # Frequency at which the sensor publishes data (Hz)
    publish_frequency: 10.0

    # Range threshold for obstacle detection (meters)
    detection_range: 10.0

    # Enable/disable sensor data processing
    enable_processing: true

Debugging Launch Files​

Verbose Output​

Use the --debug flag to get more detailed information:

ros2 launch my_package my_launch_file.py --debug

Check Launch File Syntax​

Validate your launch file without running it:

python3 my_launch_file.py

Common Launch File Patterns​

1. Robot-Specific Launch Files​

Create separate launch files for different robots or configurations:

launch/
├── robot1.launch.py
├── robot2.launch.py
├── simulation.launch.py
└── real_robot.launch.py

2. Modular Launch Files​

Break complex systems into smaller, reusable launch files:

# Include other launch files
from launch.actions import IncludeLaunchDescription
from launch.launch_description_sources import PythonLaunchDescriptionSource
from ament_index_python.packages import get_package_share_directory

def generate_launch_description():
    # Include another launch file
    navigation_launch = IncludeLaunchDescription(
        PythonLaunchDescriptionSource([
            get_package_share_directory('nav2_bringup'),
            '/launch/navigation_launch.py'
        ])
    )

    return LaunchDescription([
        navigation_launch,
        # Additional nodes...
    ])

Launch files and parameter management are essential for creating maintainable and configurable ROS 2 systems. They allow you to manage complex robotic applications with many interconnected nodes efficiently.
